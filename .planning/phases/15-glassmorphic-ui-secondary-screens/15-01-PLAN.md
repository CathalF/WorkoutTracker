# Phase 15 — Plan 01: Glassmorphic UI — Secondary Screens

## Objective
Apply the glassmorphic design system to all remaining secondary screens — ExercisePicker, TemplateManagement, Progress, Settings, and WorkoutDetail — so every screen in the app shares the same premium frosted-glass aesthetic established in phases 13–14. Three screens get extra design attention: ProgressScreen (glowing chart accents), SettingsScreen (premium control panel), and WorkoutDetailScreen (polished recap).

## Scope
- 7 tasks across 6 screens + 1 navigator
- Estimated: ~800 lines modified across ~7 files
- No new dependencies — uses glass components from phase 13
- SettingsScreen requires navigator config change (native header → custom glass header)
- ExerciseLibraryScreen is not in any navigator (orphaned code) — excluded from this plan
- Single plan — tasks execute sequentially

## Execution Context
```
src/screens/ExercisePickerScreen.tsx       — gradient, glass header, glass search, glass list items
src/screens/TemplateManagementScreen.tsx   — gradient, glass header, glass cards, 3 modals → GlassModal
src/screens/ProgressScreen.tsx             — gradient, glass header, glass chart panels, glowing accents, glass stats

src/screens/WorkoutDetailScreen.tsx        — gradient, glass header, glass stats summary, glass exercise cards
src/navigation/DashboardStackNavigator.tsx — headerShown: false for Settings
src/components/glass/*                     — (read-only) GradientBackground, GlassCard, GlassButton, GlassInput, GlassModal
src/theme/*                                — (read-only) colors, tokens, ThemeContext
```

### Task 4: SettingsScreen glassmorphic redesign
**Priority: High | Files: `src/screens/SettingsScreen.tsx`, `src/navigation/DashboardStackNavigator.tsx`**

Spotlight screen — premium control panel with individual glass card rows (385 lines + navigator change).

**How:**

1. **Navigator change**: In `DashboardStackNavigator.tsx`, change Settings screen from `headerShown: true` to `headerShown: false` so we can render a custom glass header
2. **Container**: Replace `colors.background` with `GradientBackground` wrapper
3. **Custom glass header**: Add BlurView glass header with "Settings" title and back button (same pattern as other screens):
   ```typescript
   <View style={styles.header}>
     <BlurView intensity={80} tint={isDark ? 'dark' : 'light'} style={StyleSheet.absoluteFill} />
     <View style={[StyleSheet.absoluteFill, { backgroundColor: colors.glassElevated }]} />
     <Pressable onPress={() => navigation.goBack()}>
       <Ionicons name="chevron-back" ... />
     </Pressable>
     <Text style={styles.headerTitle}>Settings</Text>
   </View>
   ```
4. **Section headers** ("Training", "Workout Reminders", etc.): Style as section labels on gradient — use `textSecondary` color, transparent background
5. **Individual setting rows → glass card rows**: Each setting (theme toggle, weekly goal, notification toggle, etc.) gets its own `GlassCard` or `glassSurface` + `glassBorder` container:
   - Setting label + control in a row
   - Clean internal layout (label left, control right)
   - Premium feel — each row is a separate frosted card
6. **Switch toggles**: Keep standard `Switch` component but ensure `trackColor` and `thumbColor` work on glass
7. **Option button grids** (weekly goal numbers, nudge days): Glass-styled option buttons — `glassSurface` + `glassBorder` for unselected, `primary` for selected
8. **Day picker buttons**: Same glass styling for unselected days, `primary` for selected
9. **Time adjuster** (+/- buttons): Glass-styled control buttons
10. **Theme**: Switch to `useThemeControl()` for `isDark`
11. Remove the `navigation.setOptions({ title: 'Settings' })` call since we're rendering a custom header

**Checkpoint:** Settings shows gradient with custom glass header. Each setting is an individual glass card row. Toggles, option buttons, and day pickers look correct on glass. Back navigation works. Both themes correct.
