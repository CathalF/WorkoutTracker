# Phase 13 — Plan 01: Glassmorphic Design System

## Objective
Build a complete glassmorphic design system with reusable components, expanded theme tokens, and dual-mode (dark + light) support. This phase creates the visual foundation — glass primitives, typography scale, spacing tokens, gradient backgrounds — that phases 14-15 will consume to redesign every screen.

## Scope
- 10 tasks across theme infrastructure and component creation
- Estimated: ~1000 lines new, ~80 lines modified across ~15 files
- 1 new dependency: `expo-blur`
- Single plan — tasks build sequentially (tokens → infrastructure → components)

### Task 4: Upgrade ThemeContext with manual theme switching
**Priority: High | Files: `src/theme/ThemeContext.tsx`, `src/theme/index.ts`**

Add support for manual theme override (`system` | `light` | `dark`). Persist the user's choice via the existing `getSetting`/`setSetting` in the settings table. Expose `themeMode` and `setThemeMode` through context.

**How:**

Expand the context to provide both colors and theme control:

```typescript
type ThemeMode = 'system' | 'light' | 'dark';

interface ThemeContextValue {
  colors: ThemeColors;
  themeMode: ThemeMode;
  setThemeMode: (mode: ThemeMode) => void;
  isDark: boolean;
}
```

In `ThemeProvider`:
1. Read initial theme mode from `getSetting('theme_mode', 'system')` using `useState` with lazy initializer
2. Use `useColorScheme()` for system preference
3. Resolve effective theme: if mode is `'system'`, use system scheme; otherwise use the explicit mode
4. On `setThemeMode`, call `setSetting('theme_mode', mode)` and update state

Update `useTheme()` to return just `colors` (backward compatible). Add a new `useThemeControl()` hook that returns the full `ThemeContextValue`.

Update `src/theme/index.ts` to export `useThemeControl`, `ThemeMode`, and `ThemeContextValue`.

**Checkpoint:** App still uses system theme by default. Calling `setThemeMode('dark')` forces dark mode. Preference survives app restart.

### Task 5: Build GradientBackground component
**Priority: High | Files: `src/components/glass/GradientBackground.tsx`**

Create a full-screen gradient background component that renders behind glass surfaces. This provides the colored backdrop that peeks through frosted glass panels.

**How:**

```typescript
import { LinearGradient } from 'expo-linear-gradient';
import { StyleSheet } from 'react-native';
import { useTheme } from '../../theme';

export function GradientBackground({ children }: { children: React.ReactNode }) {
  const colors = useTheme();
  return (
    <LinearGradient
      colors={[colors.gradientStart, colors.gradientMid, colors.gradientEnd]}
      locations={[0, 0.5, 1]}
      start={{ x: 0.1, y: 0 }}
      end={{ x: 0.9, y: 1 }}
      style={StyleSheet.absoluteFill}
    >
      {children}
    </LinearGradient>
  );
}
```

The gradient covers the full screen, diagonal from top-left to bottom-right. Children (the screen content with glass cards) render on top.

**Checkpoint:** Rendering `<GradientBackground />` shows a smooth diagonal gradient in both dark and light modes.

### Task 6: Build GlassCard component
**Priority: High | Files: `src/components/glass/GlassCard.tsx`**

The primary building block — a frosted glass container for content. Uses `BlurView` from `expo-blur` with semi-transparent background and subtle borders.

**How:**

Props interface:
```typescript
interface GlassCardProps {
  children: React.ReactNode;
  blur?: 'light' | 'medium' | 'heavy';  // default: 'medium'
  style?: StyleProp<ViewStyle>;
  padding?: keyof typeof spacing;        // default: 'base'
}
```

Implementation approach:
1. Outer `View` with `borderRadius`, `overflow: 'hidden'`, shadow styles
2. Inner `BlurView` with `intensity` mapped from `glass.blur[blur]` and `tint` based on `isDark`
3. Content wrapper `View` with `backgroundColor: colors.glassSurface`, `borderWidth: glass.borderWidth`, `borderColor: colors.glassBorder`
4. Children rendered inside the content wrapper with padding from spacing tokens

The glass border is semi-transparent white (dark mode) or semi-transparent white (light mode) — creating the characteristic glass edge highlight.

**Checkpoint:** `<GlassCard><Text>Hello</Text></GlassCard>` renders a frosted glass panel over a gradient background. Content is readable in both themes.

### Task 7: Build GlassButton component
**Priority: High | Files: `src/components/glass/GlassButton.tsx`**

Two variants: **primary** (solid electric blue) and **secondary** (glass-styled outline).

**How:**

Props interface:
```typescript
interface GlassButtonProps {
  title: string;
  onPress: () => void;
  variant?: 'primary' | 'secondary';  // default: 'primary'
  disabled?: boolean;
  icon?: string;                       // Ionicons name
  size?: 'sm' | 'md' | 'lg';          // default: 'md'
  style?: StyleProp<ViewStyle>;
}
```

**Primary variant:**
- Background: `colors.primary` (electric blue)
- Text: white
- Pressed: `colors.primaryPressed`
- Disabled: `colors.primaryDisabled` background, reduced text opacity
- Border radius from `radii.md`
- No blur effect (solid accent surface)

**Secondary variant:**
- Background: `colors.glassSurface` with `BlurView` (like GlassCard)
- Text: `colors.primary` (electric blue)
- Border: `1px solid colors.glassBorder`
- Pressed: slight opacity reduction
- Disabled: reduced opacity on entire button

**Size mapping:**
- `sm`: height 36, fontSize `typography.size.sm`, paddingHorizontal `spacing.md`
- `md`: height 44, fontSize `typography.size.base`, paddingHorizontal `spacing.base`
- `lg`: height 52, fontSize `typography.size.md`, paddingHorizontal `spacing.xl`

Optional icon renders `<Ionicons>` before the title text with `spacing.sm` gap.

Uses `Pressable` with pressed state styling.

**Checkpoint:** Both button variants render correctly. Primary is solid blue, secondary is glass. Pressed/disabled states work.

### Task 8: Build GlassInput component
**Priority: High | Files: `src/components/glass/GlassInput.tsx`**

A text input with glassmorphic styling — frosted glass background, accent-colored focus border.

**How:**

Props interface:
```typescript
interface GlassInputProps extends Omit<TextInputProps, 'style'> {
  label?: string;
  style?: StyleProp<ViewStyle>;        // container style

```

Implementation:

2. Input container: `View` with `overflow: 'hidden'`, `borderRadius: radii.md`
3. `BlurView` with `glass.blur.light` intensity (lighter blur for inputs — content must be very readable)

   - Height: 44, padding from spacing tokens
   - `typography.size.base` font size

Track focus state with `useState` + `onFocus`/`onBlur` to toggle border color.

**Checkpoint:** `<GlassInput label="Exercise" placeholder="Search..." />` renders a glass input. Focus shows blue border. Text is readable in both themes.
