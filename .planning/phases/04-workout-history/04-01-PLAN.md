# Phase 4: Workout History

## Objective
Build the workout history experience — users can browse past workouts chronologically, filter by muscle group, view full session details (exercises, sets, reps, weight), edit set values, and delete workouts.

## Execution Context
- **Phase:** 4 of 6
- **Starting state:** Working Expo app with 4 tabs (Exercises, Log Workout, History placeholder, Progress placeholder). Complete workout logging flow saves workouts + sets to SQLite. Database has `workouts` and `sets` tables with full CRUD for creation.
- **Mode:** yolo (execute without pausing for approval)
- **Parallelization:** false
- **Depends on:** Phases 1-3 completed — database singleton, typed service layer, workout/set creation functions, navigation shell with History tab (placeholder)

## Context
- **Tech stack:** React Native + Expo (SDK 54), TypeScript 5.9, React Navigation 7.1 (bottom-tabs + native-stack), expo-sqlite 16.0
- **Database tables:**
  - `workouts` — id, date (TEXT ISO), muscle_group_id (FK), notes (nullable), created_at
  - `sets` — id, workout_id (FK CASCADE), exercise_id (FK), set_number, weight (REAL), reps (INTEGER)
  - `muscle_groups` — id, name (7 groups)
  - `exercises` — id, name, muscle_group_id (FK), is_custom
- **Existing service functions to extend:**
  - `src/database/services.ts` — has `createWorkout`, `addSet`, `getWorkoutWithSets`, `deleteWorkout`, plus exercise queries
  - `getWorkoutWithSets(workoutId)` returns `WorkoutDetail` with grouped exercises and sets — reusable for detail screen
  - `deleteWorkout(workoutId)` with CASCADE delete on sets — already exists
- **Existing types:**
  - `src/types/index.ts` — `Workout`, `WorkoutSet`, `ExerciseWithSets`, `WorkoutDetail`
- **Files to modify/replace:**
  - `src/screens/HistoryScreen.tsx` — current placeholder ("View past workouts" text)
  - `src/navigation/AppNavigator.tsx` — History tab currently points to placeholder, needs stack navigator
- **Design standards (from existing screens):**
  - Primary: `#007AFF`, Success: `#34C759`, Destructive: `#FF3B30`
  - Backgrounds: `#fff` / `#F2F2F7`, Text: `#000` / `#8E8E93`
  - Border: `#C6C6C8` / `#E5E5EA`, Pressed: `#E8F0FE`
  - Cards: white bg, 12px border-radius, light border or shadow
  - Typography: 28px large title, 17px semibold headers, 16px body, 13-15px secondary
  - Spacing: 16-24px horizontal padding, 8px margin increments
  - Components: Pressable, SectionList/FlatList, Ionicons, Modal
- **Key decisions:**
  - expo-sqlite sync API (openDatabaseSync + execSync)
  - Workout date stored as ISO string
  - CASCADE delete on sets when workout deleted

## Tasks

### Task 1: Add workout listing types and service functions
**What:** Add a summary type for the history list and service functions to query workouts with filtering.
**How:**
- Add new type to `src/types/index.ts`:
  ```typescript
  interface WorkoutSummary {
    id: number;
    date: string;
    muscle_group_name: string;
    exercise_count: number;
    set_count: number;
    total_volume: number;  // sum(weight * reps)
    created_at: string;
  }
  ```
- Add functions to `src/database/services.ts`:
  ```typescript
  // Get all workouts as summaries, ordered by date DESC
  // Optional muscleGroupId filter
  getWorkouts(muscleGroupId?: number): WorkoutSummary[]

  // Update a set's weight and reps
  updateSet(setId: number, weight: number, reps: number): void

  // Delete a single set
  deleteSet(setId: number): void
  ```
- `getWorkouts` SQL: JOIN workouts + muscle_groups, subquery for exercise_count (COUNT DISTINCT exercise_id from sets), set_count (COUNT from sets), total_volume (SUM weight*reps from sets). Filter by muscle_group_id if provided. ORDER BY date DESC, created_at DESC.
- `updateSet`: simple UPDATE on sets table by id
- `deleteSet`: DELETE from sets WHERE id = ?
- All functions use `getDatabase()` singleton and parameterized queries
**Output:** `src/types/index.ts` updated with `WorkoutSummary`, `src/database/services.ts` extended with 3 new functions

### Task 3: Build the History list screen
**What:** Replace the History placeholder with a full workout history list, grouped by date, with muscle group filtering.
**How:**
- Rewrite `src/screens/HistoryScreen.tsx`:
- **Data loading:** Call `getWorkouts()` on mount and when screen focuses (use `useFocusEffect` to refresh when returning from Log Workout tab). Store results in state.
- **Date grouping:** Group workouts into sections by date. Format section headers as relative/readable dates:
  - Today → "Today"
  - Yesterday → "Yesterday"
  - Same year → "Monday, Feb 16"
  - Different year → "Monday, Feb 16, 2025"
- **Muscle group filter:** Horizontal scrollable row of filter chips at the top:
  - "All" chip (default, selected state)
  - One chip per muscle group (from `getAllMuscleGroups()`)
  - Selected chip: blue background, white text
  - Unselected chip: light gray background, dark text
  - Tapping a chip filters the list (calls `getWorkouts(muscleGroupId)`)
- **Workout card (renderItem):** Each workout shown as a card:
  - Muscle group name (bold, 17px)
  - Date/time formatted (e.g., "2:30 PM")
  - Stats row: "X exercises · Y sets · Z lbs total volume"
  - Right chevron icon indicating detail navigation
  - On tap: navigate to `WorkoutDetail` with `{ workoutId }`
- **Empty state:** When no workouts exist:
  - Large clock/history icon (64px, light gray)
  - "No workouts yet" title
  - "Complete a workout to see it here" subtitle
- **Empty filter state:** When filter has no results:
  - "No [Muscle Group] workouts" message
- **SectionList** with section headers (date labels) and workout cards as items
- **Pull-to-refresh** (optional but nice): `refreshing` state + `onRefresh` callback
- Styling consistent with existing screens (card styles from StartWorkoutScreen, section headers from ExerciseLibraryScreen)
**Output:** Fully functional history list screen with date grouping, muscle group filter chips, and navigation to detail
