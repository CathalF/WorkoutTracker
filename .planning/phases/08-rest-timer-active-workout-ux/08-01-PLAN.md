## Objective
Add a built-in rest timer that auto-starts on set completion, notifies the user when rest ends (even if backgrounded), and improve the active workout experience with exercise reordering and per-exercise notes.

## Tasks

### Task 1: Install dependencies, configure notifications, and add database migration for rest times
**What:** Install expo-notifications, expo-haptics, and expo-keep-awake. Configure the notification handler and permissions. Add a `default_rest_seconds` column to the exercises table. Add service functions for rest time settings.
**How:**
- Run `npx expo install expo-notifications expo-haptics expo-keep-awake`
- Update `app.json` to add the notifications plugin:
  ```json
  "plugins": [
    "expo-sqlite",
    ["expo-notifications", { "sounds": [] }]
  ]
  ```
- **Set up notification handler** in `App.tsx`:
  - Import `* as Notifications from 'expo-notifications'` at the top of the file
  - Before the `App` component, call:
    ```typescript
    Notifications.setNotificationHandler({
      handleNotification: async () => ({
        shouldPlaySound: true,
        shouldSetBadge: false,
        shouldShowBanner: true,
        shouldShowList: true,
      }),
    });
    ```
  - In `App()`, after `initializeDatabase()` succeeds, call a new `requestNotificationPermissions()` function:
    ```typescript
    async function requestNotificationPermissions() {
      const { status } = await Notifications.getPermissionsAsync();
      if (status !== 'granted') {
        await Notifications.requestPermissionsAsync();
      }
    }
    ```
  - Call this in the `attemptInit` function after `initializeDatabase()`, using `requestNotificationPermissions()` (fire-and-forget, don't block app load on permission result)
- **Add `default_rest_seconds` column to exercises table** in `src/database/database.ts`:
  - After the existing `CREATE TABLE IF NOT EXISTS` statements, add a migration block:
    ```typescript
    // Migration: add rest time column to exercises
    try {
      database.execSync(`ALTER TABLE exercises ADD COLUMN default_rest_seconds INTEGER`);
    } catch {
      // Column already exists — ignore
    }
    ```
  - This uses the try/catch pattern because SQLite's `ALTER TABLE ADD COLUMN` throws if the column already exists, and SQLite doesn't support `IF NOT EXISTS` for `ALTER TABLE`
  - The column is nullable — `null` means "use app default" (90 seconds)
- **Add service functions** to `src/database/services.ts`:
  ```typescript
  export const DEFAULT_REST_SECONDS = 90;

  export function getExerciseRestTime(exerciseId: number): number {
    const db = getDatabase();
    const row = db.getFirstSync<{ default_rest_seconds: number | null }>(
      'SELECT default_rest_seconds FROM exercises WHERE id = ?',
      exerciseId
    );
    return row?.default_rest_seconds ?? DEFAULT_REST_SECONDS;
  }

  export function setExerciseRestTime(exerciseId: number, seconds: number): void {
    const db = getDatabase();
    db.runSync('UPDATE exercises SET default_rest_seconds = ? WHERE id = ?', seconds, exerciseId);
  }

  export function clearExerciseRestTime(exerciseId: number): void {
    const db = getDatabase();
    db.runSync('UPDATE exercises SET default_rest_seconds = NULL WHERE id = ?', exerciseId);
  }
  ```
**Output:** Three new packages installed, notification handler configured, exercises table gains `default_rest_seconds` column, 3 new service functions + 1 constant

### Task 2: Build useRestTimer hook
**What:** Create a reusable `useRestTimer` custom hook that manages countdown state using timestamps, handles app background/foreground transitions, and exposes start/stop/adjust controls.
**How:**
- Create `src/hooks/useRestTimer.ts`:
  ```typescript
  import { useRef, useState, useEffect, useCallback } from 'react';
  import { AppState, AppStateStatus } from 'react-native';

  interface UseRestTimerOptions {
    onComplete: () => void;
  }

  interface UseRestTimerReturn {
    secondsRemaining: number;
    isRunning: boolean;
    totalDuration: number;
    startTimer: (durationSeconds: number) => void;
    stopTimer: () => void;
    adjustTime: (deltaSeconds: number) => void;
  }
  ```
- **Implementation details:**
  - Store `endTimeRef = useRef<number>(0)` — the absolute timestamp when the timer expires
  - Store `totalDurationRef = useRef<number>(0)` — the original duration (for progress bar calculation)
  - `startTimer(duration)`: sets `endTimeRef.current = Date.now() + duration * 1000`, sets `totalDuration`, sets `isRunning = true`, starts `setInterval(tick, 250)`
  - `tick()`: calculates `remaining = Math.max(0, Math.ceil((endTimeRef.current - Date.now()) / 1000))`, updates `secondsRemaining`, if remaining reaches 0 calls `stopTimer()` then `onComplete()`
  - `stopTimer()`: clears interval, sets `isRunning = false`, resets `secondsRemaining = 0`
  - `adjustTime(delta)`: adjusts `endTimeRef.current += delta * 1000`, clamps so remaining doesn't go below 1 second. If `delta < 0` and remaining would be ≤ 0, stop the timer and call `onComplete()`
  - **AppState listener** in a `useEffect`:
    - When app transitions from `inactive|background` → `active` and `isRunning` is true:
      - Immediately call `tick()` to recalculate remaining time (may have expired while backgrounded)
      - Restart the interval (it may have been suspended by the OS)
    - Store `appStateRef = useRef<AppStateStatus>(AppState.currentState)`
  - **Cleanup**: `useEffect` cleanup clears the interval on unmount
  - Use `useCallback` for `startTimer`, `stopTimer`, `adjustTime` to keep references stable
- **Return:** `{ secondsRemaining, isRunning, totalDuration, startTimer, stopTimer, adjustTime }`
- Export the hook as default
**Output:** `src/hooks/useRestTimer.ts` — self-contained timer hook with background resilience

### Task 3: Integrate rest timer UI into ActiveWorkoutScreen with set completion
**What:** Add a "complete set" checkmark button to each set row, track completed sets, auto-start the rest timer on set completion, and display a timer bar at the bottom of the screen during rest periods.

      - Schedule a notification: `scheduleRestNotification(restSeconds)`
- **Set row UI changes:**
  - Add a checkmark button to the left of each set row (before the set number), replacing the set number text:
    - Unchecked: circle outline (`checkmark-circle-outline`) in `textSecondary` color, 22px
    - Checked: filled circle (`checkmark-circle`) in `success` color, 22px
  - When a set is checked, apply subtle visual feedback:
    - Set number text gets `success` color
    - Weight/reps inputs get reduced opacity (0.6) and become non-editable (`editable={false}`)
    - The set row background gets a very subtle success tint
  - Move the set number to the right of the checkmark
- **Timer bar UI (shows when `isRunning` is true):**
  - Render a fixed-position bar at the bottom of the screen (outside the ScrollView, above tab bar area):
    - Background: `colors.surface` with top border
    - Height: ~80px
    - Layout:
      - Left: countdown text in large format `"1:30"` (32px bold, `colors.text`)
      - Center: exercise name for context ("Resting..." label, 13px secondary)
      - Right: control buttons
    - Buttons row below the countdown:
      - "-30s" button (decrease rest time)
      - "+30s" button (increase rest time)
      - "Skip" button (stops timer, primary color)
    - Progress bar: thin (3px) bar below the timer text showing elapsed/total progress. Uses `colors.primary` fill, `colors.separator` track
  - Timer bar should have a slide-up animation when appearing (use `Animated.Value` for translateY)
- **Timer completion behavior:**
  - When timer reaches 0 (via `onComplete` callback):
    - Cancel the scheduled notification
    - Trigger haptic feedback
    - Briefly show "Rest Complete!" text in the timer bar before it slides away (auto-dismiss after 1.5 seconds)
- **Notification scheduling:**
  - Create helper functions at the top of the file or in a new `src/utils/notifications.ts`:
    ```typescript
    import * as Notifications from 'expo-notifications';

    let activeNotificationId: string | null = null;

    export async function scheduleRestNotification(seconds: number) {
      // Cancel any existing scheduled notification
      if (activeNotificationId) {
        await Notifications.cancelScheduledNotificationAsync(activeNotificationId);
      }
      activeNotificationId = await Notifications.scheduleNotificationAsync({
        content: {
          title: 'Rest Complete',
          body: 'Time to start your next set!',
          sound: true,
          priority: Notifications.AndroidNotificationPriority.HIGH,
        },
        trigger: {
          type: Notifications.SchedulableTriggerInputTypes.TIME_INTERVAL,
          seconds,
          repeats: false,
        },
      });
    }

    export async function cancelRestNotification() {
      if (activeNotificationId) {
        await Notifications.cancelScheduledNotificationAsync(activeNotificationId);
        activeNotificationId = null;
      }
    }
    ```
- **Haptic feedback on completion:**
  ```typescript
  import * as Haptics from 'expo-haptics';
  // In onComplete callback:
  Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success);
  ```
- **Keep-awake:**
  - Import `useKeepAwake` from `expo-keep-awake`
  - Call `useKeepAwake()` at the top of the component to keep the screen on during the active workout
- **Import updates:** Add `getExerciseRestTime` from services, `useRestTimer` from hooks, `Animated` from react-native
**Output:** ActiveWorkoutScreen with set completion checkmarks, auto-starting rest timer, timer bar with countdown/controls, notification scheduling, haptic feedback, keep-awake
