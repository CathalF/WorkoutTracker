# Phase 5: Progress Charts

## Objective
Build per-exercise progress visualization — users can select an exercise, choose a time range, and view weight progression (line chart) and volume trends (bar chart), plus see personal records.

- **Phase:** 5 of 6
- **Starting state:** Working Expo app with 4 tabs. Exercises, Log Workout, and History are fully functional. Progress tab is a placeholder screen. Database has workouts + sets with weight/reps data ready for charting.
- **Mode:** yolo (execute without pausing for approval)
- **Parallelization:** false
- **Depends on:** Phases 1-4 completed — database singleton, typed service layer, workout/set creation, full history with edit/delete
- **Research:** Completed (05-RESEARCH.md) — using `react-native-gifted-charts` with `expo-linear-gradient` and `react-native-svg`

- **Existing types:**
  - `src/types/index.ts` — MuscleGroup, Exercise, ExerciseWithMuscleGroup, Workout, WorkoutSet, ExerciseWithSets, WorkoutDetail, WorkoutSummary
- **Files to modify/replace:**
  - `src/screens/ProgressScreen.tsx` — current placeholder ("Track your gains" text)
  - `src/database/services.ts` — add chart data query functions
  - `src/types/index.ts` — add chart data types
- **Design standards (from existing screens):**
  - Primary: `#007AFF`, Success: `#34C759`, Destructive: `#FF3B30`
  - Backgrounds: `#fff` / `#F2F2F7`, Text: `#000` / `#8E8E93`
  - Border: `#C6C6C8` / `#E5E5EA`, Pressed: `#E8F0FE`
  - Cards: white bg, 12px border-radius, light border or shadow
  - Typography: 28px large title, 17px semibold headers, 16px body, 13-15px secondary
  - Spacing: 16-24px horizontal padding, 8px margin increments
  - Components: Pressable, SectionList/FlatList, Ionicons, Modal

  - Aggregation done in SQL, not JS
  - Same-day duplicate exercises: aggregate (MAX weight, SUM volume)

## Tasks

### Task 1: Install dependencies and add chart data types + service functions
**What:** Install the charting library, add TypeScript types for chart data, and add service functions to query exercise progress, volume, and personal records from SQLite.
**How:**
- Run: `npx expo install react-native-gifted-charts expo-linear-gradient react-native-svg`
- Add new types to `src/types/index.ts`:
  ```typescript
  interface ExerciseProgressPoint {
    date: string;
    max_weight: number;
  }

  interface ExerciseVolumePoint {
    date: string;
    total_volume: number;
  }

  interface PersonalRecord {
    weight: number;
    reps: number;
    date: string;
  }

  interface VolumeRecord {
    date: string;
    total_volume: number;
  }

  interface LoggedExercise {
    id: number;
    name: string;
    muscle_group_name: string;
  }
  ```
- Add functions to `src/database/services.ts`:
  ```typescript
  // Get exercises the user has actually logged (DISTINCT exercise_id from sets)
  getLoggedExercises(): LoggedExercise[]

  // Get max weight per workout session for an exercise, ordered by date ASC
  // Optional dateFrom filter for time range
  getExerciseProgress(exerciseId: number, dateFrom?: string): ExerciseProgressPoint[]

  // Get total volume (SUM weight*reps) per workout session for an exercise
  getExerciseVolume(exerciseId: number, dateFrom?: string): ExerciseVolumePoint[]

  // Get personal records for an exercise: max weight ever, max single-session volume
  getPersonalRecords(exerciseId: number): { maxWeight: PersonalRecord | null; maxVolume: VolumeRecord | null }
  ```
- SQL queries (from research):
  - `getExerciseProgress`: SELECT w.date, MAX(s.weight) as max_weight FROM sets s JOIN workouts w ON s.workout_id = w.id WHERE s.exercise_id = ? [AND w.date >= ?] GROUP BY w.id ORDER BY w.date ASC
  - `getExerciseVolume`: SELECT w.date, SUM(s.weight * s.reps) as total_volume FROM sets s JOIN workouts w ON s.workout_id = w.id WHERE s.exercise_id = ? [AND w.date >= ?] GROUP BY w.id ORDER BY w.date ASC
  - `getPersonalRecords`: Two queries — max weight (ORDER BY s.weight DESC LIMIT 1) and max volume session (GROUP BY w.id ORDER BY volume DESC LIMIT 1)
  - `getLoggedExercises`: SELECT DISTINCT e.id, e.name, mg.name AS muscle_group_name FROM sets s JOIN exercises e ON s.exercise_id = e.id JOIN muscle_groups mg ON e.muscle_group_id = mg.id ORDER BY e.name
- All functions use `getDatabase()` singleton and parameterized queries
**Output:** Dependencies installed, `src/types/index.ts` updated with chart types, `src/database/services.ts` extended with 4 new functions

### Task 2: Build Progress screen with exercise selector and time range controls
**What:** Replace the Progress placeholder with a full screen layout: header, exercise selector dropdown, time range chip row, and weight/volume toggle.
**How:**
- Rewrite `src/screens/ProgressScreen.tsx`:
- **Header:** Custom header matching History screen pattern (paddingTop 60, 28px bold "Progress" title, white background)
- **Exercise selector:**
  - Load logged exercises via `getLoggedExercises()` on mount
  - Pressable showing currently selected exercise name + chevron-down icon
  - On press: open a Modal with FlatList of logged exercises, grouped or flat list
  - Each exercise row shows name + muscle group name subtitle
  - On select: set selectedExerciseId state, close modal
  - Default: auto-select the first logged exercise
- **Time range controls:**
  - Horizontal row of 4 chips: "1M", "3M", "6M", "All"
  - Same chip styling pattern as HistoryScreen filter chips (blue selected, gray unselected)
  - State: `timeRange` — "1M" | "3M" | "6M" | "ALL" (default "ALL")
  - On select: update timeRange state, which triggers data re-fetch
  - Compute dateFrom: 1M = 30 days back, 3M = 90 days, 6M = 180 days, ALL = undefined
- **Chart type toggle:**
  - Two-segment toggle: "Weight" | "Volume"
  - Styled as a segmented control: rounded container, selected segment gets blue background + white text
  - State: `chartType` — "weight" | "volume" (default "weight")
- **Data loading:**
  - When selectedExerciseId or timeRange changes, call `getExerciseProgress()` or `getExerciseVolume()` depending on chartType
  - Also call `getPersonalRecords()` when exercise changes
  - Store data in state: `progressData`, `volumeData`, `personalRecords`
- **Empty state (no logged exercises):**
  - Large analytics icon (Ionicons `analytics-outline`, 64px, light gray)
  - "No data yet" title
  - "Log some workouts to see your progress" subtitle
- Leave chart area as a placeholder View with "Chart goes here" text (Task 3 fills it in)
- ScrollView as the root content container (charts + PR card may exceed screen height)
**Output:** Fully functional Progress screen with exercise selection, time range, chart type toggle, and data loading wired up

### Task 3: Add chart visualizations
**What:** Integrate `react-native-gifted-charts` to render a line chart for weight progression and a bar chart for volume trends.
**How:**
- Import `LineChart` and `BarChart` from `react-native-gifted-charts`
- Replace the placeholder chart area in ProgressScreen with:
- **Weight chart (LineChart):**
  - Data: `progressData` mapped to `{ value: number, label?: string, dataPointText?: string }`
  - Format labels: show abbreviated month/day (e.g., "Jan 5") — show every Nth label to prevent overlap based on data length
  - `color: '#007AFF'` (primary blue), `dataPointsColor: '#007AFF'`
  - `thickness: 2`, `dataPointsRadius: 4`
  - `pointerConfig` for touch interaction: show tooltip with weight value on press
  - Set `yAxisLabelWidth: 50` (explicit, per research pitfalls)
  - Set `yAxisTextStyle`, `xAxisLabelTextStyle` with `fontSize: 11, color: '#8E8E93'`
  - Add `overflowTop: 30` for tooltip headroom
  - `areaChart: true` with light blue gradient fill for visual appeal
  - Chart wrapped in a white card container with 12px border-radius
  - "Max Weight (lbs)" label above chart
- **Volume chart (BarChart):**
  - Data: `volumeData` mapped to `{ value: number, label?: string }`
  - `frontColor: '#34C759'` (green, to differentiate from weight chart)
  - `barWidth: 20`, `barBorderRadius: 4`
  - Label formatting same as weight chart
  - `yAxisLabelWidth: 60` (volume numbers are larger)
  - `renderTooltip` for showing volume on bar press
  - Chart wrapped in a white card container
  - "Total Volume (lbs)" label above chart
- **Conditional rendering:** Show weight chart when `chartType === 'weight'`, volume chart when `chartType === 'volume'`
- **Data formatting with useMemo:**
  - Wrap chart data transformations in `useMemo` to avoid re-renders from recreating arrays
  - Label thinning: if data has >10 points, show every 2nd or 3rd label; if >20, show every 5th
  - Format date labels: `new Date(dateStr).toLocaleDateString('en-US', { month: 'short', day: 'numeric' })` → "Feb 16"
- **Insufficient data state:** If selected exercise has 0 data points for the time range, show "No data for this time range" message instead of chart. If only 1 data point, show it as a single point with a message "Log more sessions to see trends".
- Use `adjustToWidth: true` or calculate `spacing` based on container width and data point count to prevent overflow
**Output:** Interactive line chart (weight) and bar chart (volume) rendering real workout data
