## Tasks

### Task 1: Database — personal_records table, settings table, and service functions for dashboard and PR detection
**What:** Create two new tables (`personal_records` for storing detected PRs, `settings` for user preferences like weekly goal). Add service functions for PR detection, streak calculation, and dashboard data retrieval.
**How:**
- **Add tables to `src/database/database.ts`** in the `initializeSchema()` function, after existing CREATE TABLE statements:
  ```sql
  CREATE TABLE IF NOT EXISTS personal_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    exercise_id INTEGER NOT NULL,
    pr_type TEXT NOT NULL,
    weight REAL NOT NULL,
    reps INTEGER NOT NULL,
    date TEXT NOT NULL,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (exercise_id) REFERENCES exercises(id)
  );

  CREATE TABLE IF NOT EXISTS settings (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL
  );
  ```
  - `pr_type` is either `'weight'` or `'reps'`
  - `settings` is a simple key-value store, initially for `weekly_goal` (default "3")
- **Add new types to `src/types/index.ts`:**
  ```typescript
  export interface DetectedPR {
    exerciseId: number;
    exerciseName: string;
    prType: 'weight' | 'reps';
    weight: number;
    reps: number;
    date: string;
  }

  export interface RecentPR {
    id: number;
    exercise_name: string;
    pr_type: string;
    weight: number;
    reps: number;
    date: string;
  }

  export interface MonthlyStats {
    workoutCount: number;
    totalVolume: number;
  }
  ```
- **Add service functions to `src/database/services.ts`:**
  - **PR detection:**
    ```typescript
    export function checkForWeightPR(exerciseId: number, weight: number, reps: number): boolean
    ```
    - Query: `SELECT MAX(weight) as max_weight FROM sets WHERE exercise_id = ?`
    - Returns `true` if `weight > max_weight` (or no previous sets exist = false, first workout isn't a PR)
    ```typescript
    export function checkForRepsPR(exerciseId: number, weight: number, reps: number): boolean
    ```
    - Query: `SELECT MAX(reps) as max_reps FROM sets WHERE exercise_id = ? AND weight >= ?`
    - Returns `true` if `reps > max_reps` (and max_reps is not null — must have prior sets at this weight or above)
  - **PR storage:**
    ```typescript
    export function savePR(exerciseId: number, prType: 'weight' | 'reps', weight: number, reps: number, date: string): void
    ```
    - Insert into `personal_records`
    ```typescript
    export function getRecentPRs(limit: number = 5): RecentPR[]
    ```
    - Query: `SELECT pr.*, e.name as exercise_name FROM personal_records pr JOIN exercises e ON pr.exercise_id = e.id ORDER BY pr.created_at DESC LIMIT ?`
  - **Dashboard data:**
    ```typescript
    export function getWorkoutDaysInRange(startDate: string, endDate: string): string[]
    ```
    - Query: `SELECT DISTINCT date FROM workouts WHERE date >= ? AND date <= ? ORDER BY date`
    - Returns array of ISO date strings
    ```typescript
    export function getWeeklyStreak(weeklyGoal: number): number
    ```
    - Algorithm: iterate backwards week by week from the week before the current one (current week is incomplete, so skip it). For each week, count workouts. If count >= weeklyGoal, increment streak. Stop at first week that doesn't meet the goal.
    - Uses Monday as week start. Query per week: `SELECT COUNT(*) FROM workouts WHERE date >= ? AND date <= ?`
    - Returns the streak count (0 if the most recent complete week didn't meet the goal)
    ```typescript
    export function getMonthlyStats(year: number, month: number): MonthlyStats
    ```
    - Workout count: `SELECT COUNT(*) FROM workouts WHERE date >= ? AND date < ?` (first/last of month)
    - Total volume: `SELECT COALESCE(SUM(s.weight * s.reps), 0) FROM sets s JOIN workouts w ON s.workout_id = w.id WHERE w.date >= ? AND w.date < ?`
  - **Settings:**
    ```typescript
    export function getSetting(key: string, defaultValue: string): string
    ```
    - Query: `SELECT value FROM settings WHERE key = ?`, falls back to `defaultValue`
    ```typescript
    export function setSetting(key: string, value: string): void
    ```
    - Upsert: `INSERT OR REPLACE INTO settings (key, value) VALUES (?, ?)`
**Output:** Two new tables, 3 new types, 8 new service functions (2 PR detection, 2 PR storage/retrieval, 3 dashboard data, 2 settings). No existing code changed.

### Task 2: PR detection in ActiveWorkoutScreen with celebration modal
**What:** Detect personal records in real-time when a user completes a set, and show a celebratory modal with visual flair and haptic feedback.
**How:**
- **Modify `src/screens/ActiveWorkoutScreen.tsx`:**
- **Add PR detection state:**
  ```typescript
  const [prModalData, setPrModalData] = useState<{
    exerciseName: string;
    prType: 'weight' | 'reps';
    weight: number;
    reps: number;
  } | null>(null);
  const [sessionPRKeys, setSessionPRKeys] = useState<Set<string>>(new Set());
  ```
  - `prModalData`: when non-null, the celebration modal is visible
  - `sessionPRKeys`: tracks PRs already celebrated this session to avoid duplicate celebrations. Keys are `"${exerciseId}-weight"` or `"${exerciseId}-reps-${weight}"`.
- **Add PR detection inside `toggleSetComplete()`**, after line 325 (set marked complete) and before line 328 (rest timer start):
  ```typescript
  // PR detection
  const exerciseId = exercises[exIdx].exerciseId;
  const exerciseName = exercises[exIdx].exerciseName;
  const today = new Date().toISOString().split('T')[0];

  const weightPRKey = `${exerciseId}-weight`;
  const repsPRKey = `${exerciseId}-reps-${w}`;

  if (!sessionPRKeys.has(weightPRKey) && checkForWeightPR(exerciseId, w, r)) {
    savePR(exerciseId, 'weight', w, r, today);
    setSessionPRKeys(prev => new Set(prev).add(weightPRKey));
    setPrModalData({ exerciseName, prType: 'weight', weight: w, reps: r });
    Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success);
  } else if (!sessionPRKeys.has(repsPRKey) && checkForRepsPR(exerciseId, w, r)) {
    savePR(exerciseId, 'reps', w, r, today);
    setSessionPRKeys(prev => new Set(prev).add(repsPRKey));
    setPrModalData({ exerciseName, prType: 'reps', weight: w, reps: r });
    Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success);
  }
  ```
  - Weight PR takes priority over rep PR if both are true
  - PR detection queries the database (historical data only — current session sets aren't saved yet)
  - Session keys prevent the same PR type from triggering multiple times per exercise per workout
- **PR celebration modal** — render at the bottom of the component JSX:
  ```typescript

    <Pressable style={styles.modalOverlay} onPress={() => setPrModalData(null)}>
      <Pressable style={styles.prModalContent} onPress={() => {}}>
        {/* Trophy icon with scale animation */}
        <Animated.View style={[staticStyles.prIconContainer, { transform: [{ scale: prIconScale }] }]}>
          <Ionicons name="trophy" size={56} color={colors.warning} />
        </Animated.View>

        <Text style={styles.prTitle}>New Personal Record!</Text>
        <Text style={styles.prExerciseName}>{prModalData?.exerciseName}</Text>

        <View style={styles.prValueCard}>
          <Text style={styles.prValue}>
            {prModalData?.weight} lbs × {prModalData?.reps} reps
          </Text>
          <Text style={styles.prTypeLabel}>
            {prModalData?.prType === 'weight' ? 'Weight PR' : 'Rep PR'}
          </Text>
        </View>

        <Pressable style={styles.prDismissButton} onPress={() => setPrModalData(null)}>
          <Text style={staticStyles.prDismissText}>Keep Going!</Text>
        </Pressable>
      </Pressable>
    </Pressable>

  ```

  - `useEffect` watches `prModalData`: when it becomes non-null, animate `prIconScale` from 0 → 1.2 → 1.0 using `Animated.sequence([spring to 1.2, spring to 1.0])`
