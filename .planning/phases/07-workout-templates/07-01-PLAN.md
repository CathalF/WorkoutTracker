# Phase 7: Workout Templates & Quick Start

## Objective
Let users save completed workouts as reusable templates, start new sessions from them with one tap, see their previous performance as reference, organize templates into programs, and manage templates through a dedicated screen.

## Execution Context
- **Phase:** 7 of 11 (first phase of v2.0 milestone)
- **Starting state:** Fully functional v1.0 app with 4 tabs (Exercises, Log Workout, History, Progress). All workout logging, history, and charting features complete. Dark/light theme system in place. Database has 4 tables: muscle_groups (7 rows), exercises (~46), workouts, sets.
- **Mode:** yolo (execute without pausing for approval)
- **Parallelization:** false
- **Depends on:** Phases 1-6 completed — database singleton, typed service layer, workout logging flow (StartWorkout → ActiveWorkout → ExercisePicker), theme system with `useTheme()` + `createStyles(colors)` pattern

## Tasks

### Task 1: Add template/program tables, types, and service functions
**What:** Create 3 new database tables (programs, workout_templates, template_exercises), add TypeScript types, and implement CRUD service functions for templates and programs, plus a query to fetch the user's most recent performance per exercise.
**How:**
- Add 3 new `CREATE TABLE IF NOT EXISTS` statements to `initializeSchema()` in `src/database/database.ts`:
  ```sql
  CREATE TABLE IF NOT EXISTS programs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    created_at TEXT NOT NULL DEFAULT (datetime('now'))
  );

  CREATE TABLE IF NOT EXISTS workout_templates (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    muscle_group_id INTEGER NOT NULL,
    split_label TEXT NOT NULL,
    muscle_group_ids TEXT NOT NULL,
    program_id INTEGER,
    sort_order INTEGER NOT NULL DEFAULT 0,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (muscle_group_id) REFERENCES muscle_groups(id),
    FOREIGN KEY (program_id) REFERENCES programs(id) ON DELETE SET NULL
  );

  CREATE TABLE IF NOT EXISTS template_exercises (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    template_id INTEGER NOT NULL,
    exercise_id INTEGER NOT NULL,
    sort_order INTEGER NOT NULL,
    default_sets INTEGER NOT NULL DEFAULT 3,
    FOREIGN KEY (template_id) REFERENCES workout_templates(id) ON DELETE CASCADE,
    FOREIGN KEY (exercise_id) REFERENCES exercises(id)
  );
  ```
  - `workout_templates.muscle_group_ids` stores a JSON array string (e.g., `"[1,3]"`) of the split's muscle group IDs
  - `workout_templates.program_id` is nullable — unassigned templates have no program
  - `template_exercises.sort_order` preserves exercise ordering from the original workout
  - `template_exercises.default_sets` captures how many sets the user did for this exercise (used to pre-create empty set rows)
- Add new types to `src/types/index.ts`:
  ```typescript
  export interface Program {
    id: number;
    name: string;
    created_at: string;
  }

  export interface WorkoutTemplate {
    id: number;
    name: string;
    muscle_group_id: number;
    split_label: string;
    muscle_group_ids: number[];  // parsed from JSON
    program_id: number | null;
    sort_order: number;
    created_at: string;
  }

  export interface TemplateExercise {
    id: number;
    template_id: number;
    exercise_id: number;
    exercise_name: string;
    sort_order: number;
    default_sets: number;
  }

  export interface TemplateWithExercises extends WorkoutTemplate {
    exercises: TemplateExercise[];
  }

  export interface LastPerformanceSet {
    set_number: number;
    weight: number;
    reps: number;
  }
  ```
- Add service functions to `src/database/services.ts`:
  ```typescript
  // --- Programs ---
  createProgram(name: string): number
  getAllPrograms(): Program[]
  renameProgram(programId: number, name: string): void
  deleteProgram(programId: number): void  // templates become unassigned (ON DELETE SET NULL)

  // --- Templates ---
  createTemplate(name: string, muscleGroupId: number, splitLabel: string,
    muscleGroupIds: number[], exercises: { exerciseId: number; defaultSets: number }[],
    programId?: number): number
  getAllTemplates(): WorkoutTemplate[]  // parse muscle_group_ids JSON
  getTemplateWithExercises(templateId: number): TemplateWithExercises | null
  renameTemplate(templateId: number, name: string): void
  deleteTemplate(templateId: number): void  // CASCADE deletes template_exercises
  assignTemplateToProgram(templateId: number, programId: number | null): void
  updateTemplateSortOrder(templateId: number, sortOrder: number): void

  // --- Template exercises ---
  addTemplateExercise(templateId: number, exerciseId: number, sortOrder: number, defaultSets: number): number
  removeTemplateExercise(templateExerciseId: number): void
  updateTemplateExerciseDefaultSets(templateExerciseId: number, defaultSets: number): void

  // --- Previous performance ---
  getLastPerformance(exerciseId: number): LastPerformanceSet[]
  // Returns all sets from the most recent workout containing this exercise,
  // ordered by set_number. Query: find most recent workout_id where this exercise
  // was logged, then return all sets for that exercise in that workout.
  ```
- `getLastPerformance` SQL:
  ```sql
  SELECT s.set_number, s.weight, s.reps
  FROM sets s
  WHERE s.exercise_id = ? AND s.workout_id = (
    SELECT s2.workout_id FROM sets s2
    JOIN workouts w ON s2.workout_id = w.id
    WHERE s2.exercise_id = ?
    ORDER BY w.date DESC, w.created_at DESC
    LIMIT 1
  )
  ORDER BY s.set_number
  ```
- `getAllTemplates` must parse `muscle_group_ids` from JSON string to `number[]` using `JSON.parse()`
- All functions use `getDatabase()` singleton, parameterized queries, typed returns
- `createTemplate` inserts the template row, then inserts each exercise into `template_exercises` with incrementing `sort_order`
**Output:** `src/database/database.ts` with 3 new tables, `src/types/index.ts` with 5 new types, `src/database/services.ts` with ~14 new functions

### Task 4: Start-from-template with previous performance reference
**What:** When starting a workout from a template, pre-create the exercise entries with empty sets and show the user's most recent weight/reps as reference text beside each input field.
**How:**
- Modify `src/screens/ActiveWorkoutScreen.tsx`:
- **Detect template start:**
  - In a `useEffect` (run once on mount), check if `route.params.fromTemplate` exists
  - If it does, initialize the `exercises` state from template data:
    - For each template exercise: create an `ActiveExercise` with `exerciseId`, `exerciseName`, and `defaultSets` number of empty `{ weight: '', reps: '' }` set entries
- **Previous performance data:**
  - Add new state: `lastPerformance: Map<number, LastPerformanceSet[]>` — keyed by exerciseId
  - When initializing from a template, call `getLastPerformance(exerciseId)` for each exercise
  - Store results in the `lastPerformance` map
  - Also call `getLastPerformance()` when any exercise is added (via ExercisePicker selection) — so this reference feature works for all exercises, not just template ones
- **UI for previous performance reference:**
  - In each set row, if `lastPerformance` has data for this exercise and this set number:
    - Show a small reference text below or beside the weight input: previous weight value in secondary/tertiary color (e.g., "135" in gray)
    - Show a small reference text below or beside the reps input: previous reps value (e.g., "8" in gray)
    - Format: small text (11-12px) in `colors.textTertiary`, positioned below each input field
    - Label style: `"prev: 135 × 8"` shown as a single line below the set row, or individual values below each input
  - If no previous performance exists for this exercise, show nothing (no reference text)
  - Use a compact layout that doesn't add too much vertical space — the reference should be subtle
- **Import** `getLastPerformance` from services, `LastPerformanceSet` from types
**Output:** Starting from a template pre-loads exercises with empty sets and shows previous weight/reps as reference
