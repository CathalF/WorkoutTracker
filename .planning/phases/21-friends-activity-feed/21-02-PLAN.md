  <name>Task 1: Create FriendsContext provider and hook</name>
  <files>src/contexts/FriendsContext.tsx</files>
  <action>
Create `src/contexts/FriendsContext.tsx` following the exact same pattern as `ProfileContext.tsx` and `AuthContext.tsx`.

**Interface:**
```typescript
interface FriendsContextValue {
  friends: Friendship[];
  pendingIncoming: Friendship[];
  pendingOutgoing: Friendship[];
  friendIds: string[];
  isLoading: boolean;
  refresh: () => Promise<void>;
  sendRequest: (targetUserId: string) => Promise<void>;
  accept: (friendshipId: string) => Promise<void>;
  reject: (friendshipId: string) => Promise<void>;
  remove: (friendshipId: string) => Promise<void>;
}
```

**Implementation:**
1. Create context with `createContext<FriendsContextValue | undefined>(undefined)`.
2. `FriendsProvider` takes `{ children: ReactNode }`.
3. Use `useAuth()` to get the current user. If no user, all arrays empty and functions are no-ops.
4. `refresh` (useCallback): fetches `getFriends(user.id)`, `getPendingRequests(user.id, 'incoming')`, `getPendingRequests(user.id, 'outgoing')`, and `getFriendIds(user.id)` in parallel via `Promise.all`. Sets all state variables. Wraps in try/catch/finally for isLoading. Log errors with `console.error('Failed to refresh friends:', e)`.
5. `sendRequest`, `accept`, `reject`, `remove`: each calls the corresponding friendService function, then calls `refresh()` to update state. Wrap in try/catch, log errors.
6. Initial load: `useEffect` calls `refresh()` when user.id changes (dependency: `[user?.id]`).
7. Export `useFriends()` hook: `const ctx = useContext(FriendsContext); if (!ctx) throw new Error('useFriends must be used within FriendsProvider'); return ctx;`

Use `useCallback` for all function values to prevent unnecessary re-renders.
