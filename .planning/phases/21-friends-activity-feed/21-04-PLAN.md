**HighFiveButton.tsx** -- Reusable animated reaction button. Create in `src/components/`.

Props interface:
```typescript
interface HighFiveButtonProps {
  workoutId: string;
  count: number;
  hasReacted: boolean;
  onToggle: (workoutId: string, result: 'added' | 'removed') => void;
}
```

Implementation:
1. Uses `useRef(new Animated.Value(1))` for scale.
2. On press handler (useCallback):
   a. Fire `Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium)` (import from `expo-haptics`).
   b. Run animation: `Animated.sequence([ Animated.spring(scale, { toValue: 1.4, useNativeDriver: true, speed: 30 }), Animated.spring(scale, { toValue: 1.0, useNativeDriver: true, speed: 20 }) ]).start()`.
   c. Call `onToggle(workoutId, hasReacted ? 'removed' : 'added')` immediately for optimistic update.
   d. Call `toggleHighFive(workoutId)` from friendService. On error (`catch`), call `onToggle(workoutId, hasReacted ? 'added' : 'removed')` to revert, and `console.warn('High-five failed:', e)`.
3. Render: `Pressable` (hitSlop={8}) wrapping `Animated.View` with `transform: [{ scale }]`. Inside: `Ionicons` name `hand-left` (filled if hasReacted) or `hand-left-outline` (if not). Color: `colors.primary` if reacted, `colors.textSecondary` if not. `Text` showing count next to icon. Flex row layout with small gap.
4. Use `useTheme()` for colors.

**SocialFeedScreen.tsx** -- Overwrite stub from Plan 02.

1. `GradientBackground` wrapper. Glass header: "Activity" title centered, left icon for "Friends" (Ionicons `people-outline`, navigates to FriendList), right icon for "Search" (Ionicons `search-outline`, navigates to FriendSearch).
2. State: `entries: FeedEntry[]`, `refreshing: boolean`, `page: number`, `hasMore: boolean`. Use `useRef` for `loadingMoreRef` (boolean, NOT state -- prevents double-fire per research pitfall #3) and `friendIdsRef` (cache friend IDs to avoid stale closures).
3. On mount and on tab focus (use `useFocusEffect` from `@react-navigation/native`): fetch `getFriendIds(user.id)` from friendService, store in friendIdsRef, then call `loadPage(0, true)`.
4. `loadPage(pageNum: number, replace: boolean)`:
   - `const PAGE_SIZE = 20;`
   - `const from = pageNum * PAGE_SIZE; const to = from + PAGE_SIZE - 1;`
   - `const data = await getFeedEntries(friendIdsRef.current, from, to);`
   - If replace, set entries to data. Else append: `setEntries(prev => [...prev, ...data])`.
   - `setHasMore(data.length === PAGE_SIZE)`. Set page.
5. Pull-to-refresh: `onRefresh` -> re-fetch friendIds, then `loadPage(0, true)`. Set/unset refreshing.
6. Infinite scroll: `onEndReached` with guard: `if (loadingMoreRef.current || !hasMore) return; loadingMoreRef.current = true; await loadPage(page + 1, false); loadingMoreRef.current = false;`. `onEndReachedThreshold={0.3}`.

7. **FeedCard** (inline component inside SocialFeedScreen or extracted at bottom of file):
   - Wrap in `GlassCard`. Three sections:
   - **Header row**: avatar (Image 40x40 circle, or Ionicons person-circle placeholder), display_name (bold), relative date. Tap avatar/name -> `navigation.navigate('UserProfile', { userId: entry.user_id })`.
   - **Body**: Muscle group name (derived from muscle_group_id -- use a simple mapping or show the ID; ideally map to name via a local lookup). Stats row: exercise_count exercises, set_count sets, total_volume formatted with commas + "lbs". Duration if available.
   - **PR badge**: If `entry.has_pr` is true, show `Ionicons` `trophy` icon (size 16) with "PR" text in `colors.primary`, positioned in header or body.
   - **Footer row**: `HighFiveButton` component with count, hasReacted, onToggle.
   - Tap on card body (not avatar, not high-five) -> `navigation.navigate('FriendWorkoutDetail', { workoutId: entry.id, ownerUserId: entry.user_id })`.

8. `handleHighFiveToggle(workoutId: string, result: 'added' | 'removed')`: Update entries state optimistically -- find entry by workoutId, flip `user_has_high_fived`, increment/decrement `high_five_count`. Use `setEntries(prev => prev.map(e => e.id === workoutId ? { ...e, user_has_high_fived: result === 'added', high_five_count: e.high_five_count + (result === 'added' ? 1 : -1) } : e))`.

9. `ListEmptyComponent`:
   - If `friendIdsRef.current.length === 0` (no friends): Ionicons `people-outline` large icon + "Add friends to see their workouts" text + `GlassButton` "Find Friends" navigating to FriendSearch.
   - If friendIds exist but entries empty: "Your friends haven't logged any workouts yet" text.
10. `ListFooterComponent`: `ActivityIndicator` if `loadingMoreRef.current` (use a separate `loadingMoreVisible` state just for UI rendering, toggled alongside the ref).

11. Helper: `formatRelativeDate(dateString: string): string` -- at top of file. Returns "Just now" (<1min), "Xm ago" (<60min), "Xh ago" (<24h), "Yesterday", or formatted date (e.g., "Feb 28").

12. Use `useAuth()` for user.id. Use `useNavigation<NativeStackNavigationProp<SocialStackParamList>>()`.
