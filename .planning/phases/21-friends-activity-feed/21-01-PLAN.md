**1. Create `src/lib/supabase-social-schema.sql`** -- a reference SQL file (matching the existing `supabase-profiles.sql` pattern) containing:

a) **`private` schema**: `CREATE SCHEMA IF NOT EXISTS private;`

b) **`friendships` table**: columns `id` (uuid PK DEFAULT gen_random_uuid()), `requester_id` (uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE), `addressee_id` (uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE), `status` (text NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'accepted', 'rejected'))), `created_at` (timestamptz NOT NULL DEFAULT now()), `updated_at` (timestamptz NOT NULL DEFAULT now()). Add UNIQUE(requester_id, addressee_id). Add indexes: `idx_friendships_requester` on requester_id, `idx_friendships_addressee` on addressee_id, `idx_friendships_status` on status.

c) **RLS on friendships**: Enable RLS. SELECT policy `friendships_select`: user can see rows where `requester_id = (SELECT auth.uid()) OR addressee_id = (SELECT auth.uid())`. INSERT policy `friendships_insert`: `requester_id = (SELECT auth.uid())`. UPDATE policy `friendships_update`: `requester_id = (SELECT auth.uid()) OR addressee_id = (SELECT auth.uid())`.

d) **`private.are_friends` helper function**: SECURITY DEFINER, STABLE, LANGUAGE sql. Takes (user_a uuid, user_b uuid) RETURNS boolean. Checks: `SELECT EXISTS (SELECT 1 FROM public.friendships WHERE status = 'accepted' AND ((requester_id = user_a AND addressee_id = user_b) OR (requester_id = user_b AND addressee_id = user_a)))`.

e) **`is_hidden` column on workouts**: `ALTER TABLE public.workouts ADD COLUMN IF NOT EXISTS is_hidden boolean NOT NULL DEFAULT false;`

f) **Workouts friends SELECT policy**: `workouts_friends_select` on public.workouts FOR SELECT TO authenticated. USING: `user_id = (SELECT auth.uid()) OR (NOT is_hidden AND private.are_friends((SELECT auth.uid()), user_id))`. Include a SQL comment: `-- NOTE: If an existing owner-only SELECT policy exists on workouts, drop it first before adding this one.`

g) **`high_five_reactions` table**: `id` (uuid PK DEFAULT gen_random_uuid()), `workout_id` (uuid NOT NULL REFERENCES public.workouts(id) ON DELETE CASCADE), `reactor_user_id` (uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE), `created_at` (timestamptz NOT NULL DEFAULT now()). UNIQUE(workout_id, reactor_user_id). Indexes: `idx_highfives_workout` on workout_id, `idx_highfives_reactor` on reactor_user_id.

h) **RLS on high_five_reactions**: SELECT: `reactor_user_id = (SELECT auth.uid()) OR EXISTS (SELECT 1 FROM public.workouts w WHERE w.id = workout_id AND (w.user_id = (SELECT auth.uid()) OR private.are_friends((SELECT auth.uid()), w.user_id)))`. INSERT: `reactor_user_id = (SELECT auth.uid()) AND EXISTS (SELECT 1 FROM public.workouts w WHERE w.id = workout_id AND private.are_friends((SELECT auth.uid()), w.user_id))`. DELETE: `reactor_user_id = (SELECT auth.uid())`.

i) **`push_tokens` table**: `user_id` (uuid PK REFERENCES auth.users(id) ON DELETE CASCADE), `token` (text NOT NULL), `updated_at` (timestamptz NOT NULL DEFAULT now()). RLS: FOR ALL TO authenticated, USING `user_id = (SELECT auth.uid())`, WITH CHECK `user_id = (SELECT auth.uid())`.

  <action>
Create `src/services/friendService.ts` following the same pattern as `profileService.ts`. Import `supabase` from `../lib/supabase` and types from `../types`.

**Friend Discovery:**
- `searchProfiles(query: string): Promise<UserProfile[]>` -- `.from('profiles').select('id, display_name, avatar_url, bio').ilike('display_name', '%${query.trim()}%').limit(20)`. Trim the query.

**Friendship CRUD:**
- `sendFriendRequest(addresseeId: string): Promise<void>` -- Get current user via `supabase.auth.getUser()`, INSERT into friendships with `{ requester_id: user.id, addressee_id: addresseeId }`.
- `acceptFriendRequest(friendshipId: string): Promise<void>` -- UPDATE status to 'accepted', set updated_at.
- `rejectFriendRequest(friendshipId: string): Promise<void>` -- UPDATE status to 'rejected', set updated_at.
- `removeFriend(friendshipId: string): Promise<void>` -- DELETE from friendships by id.
- `getFriends(userId: string): Promise<Friendship[]>` -- SELECT from friendships WHERE status='accepted' AND (requester_id=userId OR addressee_id=userId). For each row, determine the "other" user ID (if requester_id === userId, other is addressee_id, vice versa). Batch-fetch profiles for all other IDs via `.from('profiles').select('*').in('id', otherIds)`. Attach profile to each friendship object.
- `getPendingRequests(userId: string, direction: 'incoming' | 'outgoing'): Promise<Friendship[]>` -- SELECT WHERE status='pending'. For incoming: addressee_id=userId. For outgoing: requester_id=userId. Attach the other user's profile using the same batch-fetch pattern.
- `getFriendIds(userId: string): Promise<string[]>` -- Helper that returns array of accepted friend user IDs. Calls getFriends internally or does a leaner query selecting only the IDs.

**Activity Feed:**
- `getFeedEntries(friendIds: string[], from: number, to: number): Promise<FeedEntry[]>` -- If friendIds is empty, return []. Query workouts table with `.in('user_id', friendIds)`, `.eq('is_hidden', false)`, `.is('deleted_at', null)`, `.order('created_at', { ascending: false })`, `.range(from, to)`. Select workout columns plus joined profile data. For high-five counts and user_has_high_fived, either use a Supabase join or do a secondary query. Keep it pragmatic -- two queries is fine if the join is complex. Return mapped FeedEntry[].
- `getFriendWorkoutDetail(workoutId: string): Promise<FriendWorkoutDetail | null>` -- Fetch single workout with exercises and sets from Supabase. Join with profiles for display_name/avatar_url. Join with exercises and sets tables. Return null if not found (error code PGRST116) or RLS denies access.

**Privacy:**
- `setWorkoutHidden(workoutId: string, hidden: boolean): Promise<void>` -- UPDATE workouts SET is_hidden=hidden WHERE id=workoutId. Only works for own workouts per RLS.

**Reactions:**
- `toggleHighFive(workoutId: string): Promise<'added' | 'removed'>` -- Get current user. Check existing reaction via `.maybeSingle()`. If exists, DELETE and return 'removed'. If not, INSERT and return 'added'. Wrap in try/catch -- on error, throw so caller can handle.

**Push:**
- `savePushToken(token: string): Promise<void>` -- Get current user. UPSERT into push_tokens with `{ user_id: user.id, token, updated_at: new Date().toISOString() }`.

All functions: throw on Supabase errors (`if (error) throw error`), use `supabase.auth.getUser()` for current user ID. Keep the file under 400 lines. Use section comments to separate concerns.
