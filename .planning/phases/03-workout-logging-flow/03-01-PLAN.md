# Phase 3: Workout Logging Flow

Build the complete workout logging experience — muscle group selection, exercise picker, and set logging with fast entry UI — so users can start a workout, pick exercises, log sets/reps/weight, and save the completed workout to SQLite.

## Tasks

### Task 1: Add workout & set types and service functions
**What:** Extend the type definitions and database service layer with functions for creating, reading, and managing workouts and sets.
**How:**
- Add new types to `src/types/index.ts`:
  ```typescript
  // For displaying a workout with all its exercises and sets
  interface ExerciseWithSets {
    exercise_id: number;
    exercise_name: string;
    sets: WorkoutSet[];
  }

  interface WorkoutDetail extends Workout {
    muscle_group_name: string;
    exercises: ExerciseWithSets[];
  }
  ```
- Add functions to `src/database/services.ts`:
  ```typescript
  // Create a new workout, returns the new workout ID
  createWorkout(date: string, muscleGroupId: number, notes?: string): number

  // Add a set to a workout, returns the new set ID
  addSet(workoutId: number, exerciseId: number, setNumber: number, weight: number, reps: number): number

  // Get a workout with all its exercises and sets (grouped by exercise)
  getWorkoutWithSets(workoutId: number): WorkoutDetail | null

  // Delete an entire workout (CASCADE deletes sets)
  deleteWorkout(workoutId: number): void
  ```
- All functions use `getDatabase()` singleton, parameterized queries, typed returns
- `createWorkout` uses today's date in ISO format if not provided
- `getWorkoutWithSets` joins workouts → sets → exercises, groups sets by exercise
**Output:** `src/types/index.ts` updated, `src/database/services.ts` extended with workout/set CRUD

### Task 6: Implement workout save and completion flow
**What:** Wire up the "Finish Workout" button to save all workout data to SQLite and return to the start screen.
**How:**
- In `ActiveWorkoutScreen`, the "Finish Workout" handler:
  1. **Validate:** At least one exercise with at least one set that has weight > 0 and reps > 0
     - If validation fails, show an Alert with a message like "Add at least one complete set to save"
  2. **Save to DB:**
     - Call `createWorkout(todayDate, muscleGroupId, null)` → get `workoutId`
     - For each exercise, for each set with valid weight+reps:
       - Call `addSet(workoutId, exerciseId, setNumber, weight, reps)`
     - Skip empty/zero sets (don't save incomplete entries)
  3. **Show success:** Brief Alert or inline success message — "Workout saved!"
  4. **Navigate back:** Reset the stack navigation to StartWorkout screen
     - Use `navigation.reset({ index: 0, routes: [{ name: 'StartWorkout' }] })` to clear the stack
- Also add a **discard workout** option:
  - Back button or "Cancel" in header — if exercises have been added, show confirmation Alert ("Discard workout?")
  - If confirmed, navigate back to StartWorkout without saving
- Handle edge cases:
  - Empty string inputs treated as 0
  - Parse weight/reps as numbers before saving (parseFloat for weight, parseInt for reps)
**Output:** Complete save flow — workout data persisted to SQLite, navigation resets cleanly
