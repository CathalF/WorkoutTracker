# Phase 14 — Plan 01: Glassmorphic UI — Core Screens

## Objective
Apply the glassmorphic design system (phase 13) to the four core screens — Dashboard, Start Workout, History, and Active Workout — plus the tab bar. Every opaque `surface` card becomes a frosted `GlassCard`, solid backgrounds become `GradientBackground`, and modals become `GlassModal`. The result: the app's most-used screens feel like a premium, polished fitness product.

## Scope
- 8 tasks across 5 files (4 screens + navigator)
- Estimated: ~1000 lines modified across ~5 files
- No new dependencies — uses glass components from phase 13
- ActiveWorkoutScreen (1506 lines) split across 3 tasks for manageability
- Single plan — tasks can execute sequentially

## Tasks

### Task 1: Glass tab bar styling
**Priority: High | Files: `src/navigation/AppNavigator.tsx`**

Add a frosted glass background to the tab bar using `BlurView` and theme-aware glass tokens.

**How:**

Update the `Tab.Navigator` `screenOptions` to apply glass styling to the tab bar:

```typescript
import { BlurView } from 'expo-blur';

// In screenOptions:
tabBarStyle: {
  position: 'absolute',
  borderTopWidth: StyleSheet.hairlineWidth,
  borderTopColor: colors.glassBorder,
  elevation: 0,
  backgroundColor: 'transparent',
},
tabBarBackground: () => (
  <BlurView
    intensity={80}
    tint={isDark ? 'dark' : 'light'}
    style={[StyleSheet.absoluteFill, { backgroundColor: colors.glassSurface }]}
  />
),
```

Switch from `useTheme()` to `useThemeControl()` to get `isDark`.

**Checkpoint:** Tab bar has frosted glass appearance. Icons and labels remain visible and correctly colored. Tab bar floats over screen content.

### Task 2: DashboardScreen glassmorphic redesign
**Priority: High | Files: `src/screens/DashboardScreen.tsx`**

Convert the Dashboard to use gradient background, glass cards, and glass header.

**How:**

1. **Container**: Replace `backgroundColor: colors.background` with `GradientBackground` wrapping content
2. **Header**: Replace solid `backgroundColor: colors.surface` with transparent background + `BlurView`. Use `useThemeControl()` for `isDark`
3. **Weekly overview card**: Replace `backgroundColor: colors.surface` card with `GlassCard`
4. **Stat cards** (streak, workouts): Replace surface View with `GlassCard`
5. **Volume card**: Replace surface View with `GlassCard`
6. **PR rows**: Replace surface View with `GlassCard` styling (glassSurface background, glassBorder)
7. **Empty state**: Style over gradient (transparent background)
8. Remove `borderWidth: StyleSheet.hairlineWidth` and `borderColor: colors.border` from cards (GlassCard handles borders)
9. Replace hardcoded font sizes/weights with typography tokens where reasonable (headers, labels)

Key pattern for headers across all screens:
```typescript
<View style={styles.header}>
  <BlurView intensity={80} tint={isDark ? 'dark' : 'light'} style={StyleSheet.absoluteFill} />
  <View style={[StyleSheet.absoluteFill, { backgroundColor: colors.glassElevated }]} />
  {/* header content */}
</View>
```

**Checkpoint:** Dashboard shows gradient background with frosted glass cards. Stats, weekly overview, and PR rows all render as glass panels. Header has glass blur. Content is readable in both themes.
