## Tasks

### Task 1: Define TypeScript types
**What:** Create shared type definitions for all database entities.
**How:**
- Create `src/types/index.ts`
- Define interfaces:
  ```typescript
  interface MuscleGroup {
    id: number;
    name: string;
  }

  interface Exercise {
    id: number;
    name: string;
    muscle_group_id: number;
    is_custom: number; // 0 or 1 (SQLite boolean)
  }

  // Also define a joined type for display:
  interface ExerciseWithMuscleGroup extends Exercise {
    muscle_group_name: string;
  }

  interface Workout {
    id: number;
    date: string;
    muscle_group_id: number;
    notes: string | null;
    created_at: string;
  }

  interface WorkoutSet {
    id: number;
    workout_id: number;
    exercise_id: number;
    set_number: number;
    weight: number;
    reps: number;
  }
  ```
- Export all types
**Output:** `src/types/index.ts` with typed interfaces for all DB entities

### Task 2: Build database service layer
**What:** Create a service module with typed query functions for exercises and muscle groups. Also include workout/set query stubs for Phase 3.
**How:**
- Create `src/database/services.ts`
- Implement exercise & muscle group queries:
  ```typescript
  // Muscle groups
  getAllMuscleGroups(): MuscleGroup[]

  // Exercises
  getExercisesByMuscleGroup(muscleGroupId: number): Exercise[]
  getAllExercises(): ExerciseWithMuscleGroup[]
  searchExercises(query: string): ExerciseWithMuscleGroup[]
  addCustomExercise(name: string, muscleGroupId: number): number  // returns new ID
  deleteCustomExercise(exerciseId: number): void  // only allow deleting custom exercises
  ```
- All functions use `getDatabase()` singleton and return typed results
- Use parameterized queries (no string interpolation) for safety
- Search uses `LIKE '%query%'` with case-insensitive matching
- `addCustomExercise` sets `is_custom = 1`
- `deleteCustomExercise` checks `is_custom = 1` before deleting
**Output:** `src/database/services.ts` with typed, safe query functions

### Task 3: Build Exercise Library screen — grouped list
**What:** Replace the placeholder with a real Exercise Library screen showing exercises grouped by muscle group. This screen will be accessible from the Home tab.
**How:**
- Create `src/screens/ExerciseLibraryScreen.tsx`
- UI layout:
  - Screen title: "Exercise Library"
  - Use `SectionList` with muscle groups as section headers
  - Each section header shows the muscle group name and exercise count
  - Each row shows the exercise name (and a "Custom" badge if `is_custom = 1`)
  - Tapping a muscle group header expands/collapses that section (accordion-style)
- Load data on mount using service functions from Task 2
- Style consistently with existing app (white background, `#007AFF` accent)
- Add a floating action button (FAB) or header button for "Add Exercise" (wired in Task 6)
**Output:** Working Exercise Library screen with grouped exercise list

### Task 4: Add navigation to Exercise Library
**What:** Wire the Exercise Library screen into the app navigation.
**How:**
- Update `src/navigation/AppNavigator.tsx`:
  - Replace the Home tab placeholder with ExerciseLibraryScreen, OR
  - Better approach: Create a stack navigator for the Home tab that includes HomeScreen and ExerciseLibraryScreen
  - Add a "Browse Exercises" button/card on HomeScreen that navigates to ExerciseLibraryScreen
- Alternatively, if the tab layout feels more natural: rename the Home tab to "Exercises" and use ExerciseLibraryScreen directly as the tab screen
- Keep the navigation intuitive — the exercise library should be very easy to find (1 tap max)
**Output:** Exercise Library reachable from tab navigation

### Task 5: Add search and filter
**What:** Add a search bar to the Exercise Library screen for finding exercises by name.
**How:**
- Add a `TextInput` search bar at the top of ExerciseLibraryScreen (sticky, above the SectionList)
- On text change, call `searchExercises(query)` from the service layer
- When search is active, show flat filtered results instead of grouped sections
- When search is cleared, return to grouped view
- Add debounce (300ms) to avoid excessive queries while typing
- Show "No results" empty state when search returns nothing
**Output:** Working search/filter on Exercise Library screen

### Task 6: Add custom exercise support
**What:** Allow users to add their own exercises from the Exercise Library screen.
**How:**
- Create a simple modal/dialog for adding a custom exercise:
  - Text input for exercise name
  - Picker/dropdown to select muscle group
  - "Add" and "Cancel" buttons
- Trigger the modal from the FAB or header button added in Task 3
- On submit: call `addCustomExercise(name, muscleGroupId)` from service layer
- Validate: exercise name not empty, muscle group selected
- After adding, refresh the exercise list to show the new exercise
- Custom exercises show a "Custom" badge in the list
- Allow deleting custom exercises (long press or swipe) — only custom ones, not seeded exercises
**Output:** Users can add and delete custom exercises
